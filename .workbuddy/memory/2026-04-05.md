## Phase 6: 审批交互系统（Step 2 完成）✅

### 后端改动
- **`domain/models.py`** — SelfImprovementStatus 新增 `PENDING_APPROVAL` + `REJECTED` 枚举值；SelfImprovementJob 新增 `approval_requested_at` / `approval_edits_summary` / `approval_reason` 字段
- **`main.py`** — 新增 3 个审批 API 端点：
  - `GET /self-improvement/pending` — 查询当前待审批 Job
  - `POST /self-improvement/{job_id}/approve` — 批准（推进到 VERIFYING）
  - `POST /self-improvement/{job_id}/reject` — 拒绝（标记 REJECTED + 记录原因）
- **`service.py`** — PATCHING 阶段后插入 PENDING_APPROVAL 门控（不再直接进入 VERIFYING）；tick_job 中 PENDING_APPROVAL 状态不自动推进；新增 `_build_edits_summary()` 辅助函数

### 前端改动
- **`lib/api.ts`** — status 联合类型新增 pending_approval/rejected；Job 类型新增 3 个审批字段；新增 approve/reject/fetchPending API 函数
- **`components/ApprovalPanel.tsx`** — 全新审批面板组件：
  - 审批摘要信息（目标/原因/方案/补丁摘要）
  - 触碰文件列表
  - 可折叠 Diff 展示（精简版，三种编辑类型）
  - 批准按钮（大主操作）+ 拒绝按钮（带原因表单）
  - 批准/拒绝后展示结果状态
- **`StatusPanel.tsx`** — 当 job.status === "pending_approval" 时渲染 ApprovalPanel；新增 onApprovalDecision prop
- **`App.tsx`** — 新增 handleApprovalDecision 回调函数，传递给 StatusPanel → OverviewPanel 链路；状态映射补全 pending_approval/rejected
- **`HistoryPanel.tsx`** — 状态映射补全
- **`index.css`** — 新增 ~280 行 CSS（approval-panel / approval-summary / approval-files / approval-diff / approval-actions / reject-form）

### 核心流程
1. 数字人自编程完成 apply 后 → 状态变为 `pending_approval`
2. 前端 StatusPanel 检测到该状态 → 渲染 ApprovalPanel 替代 SelfImprovementPanel
3. 用户查看变更内容（文件列表 + Diff）→ 点击批准或拒绝
4. 批准 → 后端推进到 VERIFYING（后续自动验证）；拒绝 → REJECTED 终态
