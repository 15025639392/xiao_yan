## Phase 9: 情绪→表达风格深层映射系统 ✅

### 核心设计：让数字人的情绪真正影响说话方式

**问题**：Phase 7 建了完整的情绪引擎（12 种情绪 × 5 级强度 + 衰减 + 性格调节），但情绪只变成一句 system prompt 描述（"你现在比较开心"），并没有告诉 LLM **怎么因为情绪改变说话方式**。结果：用户调了半天性格参数，发现数字人开心和难过时说话感觉差不多。

**解决方案**：ExpressionStyleMapper — 将情绪状态转化为具体的"表达风格指令"

### 新增文件（后端 1 个 + 测试 1 个）

**后端核心：**

- `persona/expression_mapper.py` — 表达风格映射器（~550 行）：
  - **6 个表达维度枚举**：
    - `ResponseVolume` (5 级): very_brief → verbose → very_verbose
    - `EmojiLevel` (5 级): never → rarely → sometimes → often → frequently
    - `SentencePattern` (5 种): fragmented / short_direct / balanced / exclamatory / elaborate
    - `PunctuationStyle` (5 种): minimal / loose / standard / energetic / dramatic
    - `ToneModifier` (6 种): flat / gentle / playful / intense / hesitant / sarcastic
  - **12 种情绪完整映射表**：每种情绪定义基础风格 + 强烈模式(intense_instructions)
    - 正面组(JOY/GRATEFUL/PROUD/SURPRISE/ENGAGED)：话量偏多、emoji 丰富、句式活泼
    - 负面组(SADNESS/LONELY/ANGER/FRUSTRATED/FEAR/DISGUST)：话量减少、emoji 几乎不用、短句/省略
    - CALM：正常、温和
  - **ExpressionStyleMapper 类**：
    - `map_from_state(state)` → ExpressionStyleOverride（核心映射）
    - 双情绪融合：主+次要情绪按权重叠加
    - 性格调节因子：高神经质放大表达幅度，高宜人性缓和负面表达，高外向性增强正面表达，高尽责性保持条理
    - `build_style_prompt(override)` → 自然语言指令字符串
    - `build_full_expression_guide(state, base_style)` → 完整表达指南
  - 辅助方法：_amplify_emoji(提升 emoji 等级) / _blend_emoji(取较高者)
