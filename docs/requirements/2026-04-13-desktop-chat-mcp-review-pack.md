## 4) 每项决策评估

### D1 MCP 协议范围

- 方案 A 优势：实现面更小，便于先打通核心链路。
- 方案 B 风险：连接治理复杂，超时和重连策略显著增量。
- 建议：首期采用 A，二期再扩展远程协议。

### D2 审批策略

- 方案 A 优势：与当前 capability 风险模型一致，审计可复用。
- 方案 B 风险：高风险工具调用不可控，合规风险高。
- 建议：采用 A，并记录审批拒绝的降级路径。

### D3 默认白名单

- 方案 A 优势：用户开箱可用，且能力边界明确。
- 方案 B 风险：初次接入成本高，影响可用性验证。
- 建议：采用 A，同时保留关闭默认白名单开关。
