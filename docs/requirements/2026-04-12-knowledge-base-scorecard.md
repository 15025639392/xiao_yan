# 需求质量评分卡 v2（100 分）

> 用途：将“是否能进入下一阶段”从主观判断改为量化门禁。  
> 评分频率：每轮收尾至少更新一次（轻量模式可在阶段收尾更新）。

- 当前总分：99

## 1) 评分维度

### A. 完整性（0-25）

- 0-5：目标与成功指标是否明确
- 0-5：范围与非范围是否明确
- 0-5：依赖与约束是否明确
- 0-5：待确认问题是否列全并标状态
- 0-5：关键风险是否识别

### B. 可执行性（0-25）

- 0-5：任务是否可独立执行
- 0-5：owner 是否明确
- 0-5：优先级与顺序是否明确
- 0-5：依赖与阻塞条件是否明确
- 0-5：工时估算是否合理

### C. 可验收性（0-25）

- 0-5：Done 标准是否可验证
- 0-5：验收用例是否测试化（Given/When/Then）
- 0-5：验收覆盖功能与回归
- 0-5：验收覆盖性能或兼容（若适用）
- 0-5：验证方式是否可执行（命令或步骤）

### D. 风险覆盖（0-25）

- 0-5：技术风险是否有缓解方案
- 0-5：上线风险是否有门禁
- 0-5：回滚触发条件是否明确
- 0-5：回滚路径是否可执行
- 0-5：上线后观测指标是否明确

## 2) 门禁阈值

- 轻量模式：总分 >= 75，且无未处置 P0 风险
- 标准模式：总分 >= 80，且关键未知项均有 owner 和截止时间
- 发布级模式：总分 >= 85，且迁移/回滚/观测项完整

## 3) 评分记录模板

- 模式：标准
- A. 完整性：25
- B. 可执行性：25
- C. 可验收性：25
- D. 风险覆盖：24
- 总分：99
- 是否通过当前门禁：是
- 不通过时补齐项（owner + 截止时间）：无
- 风险补充：已建立 reset-first 评估口径且当前 alerts=none；但样本充足性尚未达门槛，需在观察窗口结束后再做最终放量结论。
