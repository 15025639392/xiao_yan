# 桌面端控制台 UI 重做设计

日期：2026-04-05

## 1. 目标

将现有桌面端界面重做为面向长期使用的中文控制台界面。

这次重做不改变后端接口和核心数据结构，重点调整以下三件事：

- 信息架构：让用户更快判断“小晏现在在做什么”
- 交互语义：把混杂的中英文标签统一为中文工作语境
- 视觉表达：从陪伴型落地页风格切换到有温度的控制台风格

## 2. 设计方向

采用 `Mission Control` 方案，定位为“有温度的控制台”：

- 不是冷冰冰的监控后台
- 也不是情绪优先的陪伴页
- 而是以效率优先、角色气质次之的桌面工作台

推荐这个方向的原因：

- 当前产品核心是持续操控和状态观察，而不是一次性展示
- 最近功能新增集中在晨间计划和自我修复，界面需要更强状态可见性
- 桌面端更需要结构清晰和视线引导，而不是大段叙事氛围

## 3. 信息架构

### 3.1 顶部指挥栏

顶部区域从原有 Hero 卡片改为 `指挥栏`，负责 3 秒内完成总体判断。

主要信息：

- 在线状态
- 当前阶段
- 当前焦点目标
- 快捷操作
- 轮询与同步状态

主要操作：

- `唤醒`
- `休眠`

设计原则：

- 不使用抒情性大段文案
- 不暴露原始字段名
- 让用户一眼知道系统是否在线、当前是否忙碌、是否需要进一步操作

### 3.2 中部主工作区

中部采用双栏布局：

- 左侧：`对话控制台`
- 右侧：`状态侧栏`

左侧承担主工作流，因为聊天是最高频入口。右侧承担辅助判断工作，让用户在交互时能持续感知当前状态。

### 3.3 底部目标看板

底部独立承接 `目标链` 和 `独立目标`。

目标区域不再作为普通列表，而是作为带有任务推进感的看板出现。这样可以让用户明确知道：

- 当前有哪些目标
- 各个目标处于什么状态
- 某条目标链推进到了哪一代

## 4. 组件结构

### 4.1 指挥栏

指挥栏拆成 4 个职责清晰的模块：

- `在线状态`
- `当前阶段`
- `当前专注目标`
- `快捷操作`

字段映射规则：

- `mode` 映射为中文在线状态
- `focus_mode` 映射为中文阶段名称
- `focusGoalTitle` 映射为“当前专注目标”
- `Wake / Sleep` 全部改为 `唤醒 / 休眠`

### 4.2 对话控制台

对话区改为真正的工作界面，而非普通消息列表。

结构包括：

- 顶部上下文条：显示在线态、阶段或最近系统动作
- 中部消息流：用视觉层级区分“你”和“小晏”
- 底部输入区：输入框、发送按钮、发送状态提示

设计原则：

- 不使用 `You:`、`Xiao Yan:` 这类英文前缀
- 通过标签、对齐和气泡风格区分消息身份
- 让聊天区同时具备“对话”与“命令输入”两种感受

### 4.3 状态侧栏

右侧堆叠三张卡片：

- `当前状态`
- `内在世界`
- `自我叙事`

职责边界：

`当前状态`：

- 模式
- 阶段
- 当前想法
- 最近动作
- 今日计划
- 自我修复任务

`内在世界`：

- 时间感
- 能量
- 情绪
- 专注张力
- 当前步骤
- 最近事件

`自我叙事`：

- 近期记忆
- 自我记录

### 4.4 目标看板

目标区改造成 `目标看板`：

- 目标链按时间轴或推进链展示
- 独立目标单独成组
- 目标操作全部中文化

统一按钮文案：

- `暂停`
- `恢复`
- `完成`
- `放弃`

## 5. 语言系统

整个桌面端界面统一使用中文。

要求：

- 不再混用英文标题与中文正文
- 不直接显示技术字段名
- 英文状态枚举必须转换为中文语义
- 空状态、错误文案、按钮文案、标签文案全部中文化

示例：

- `Conversation` -> `对话控制台`
- `Current State` -> `当前状态`
- `Inner World` -> `内在世界`
- `Self Narrative` -> `自我叙事`
- `Goals` -> `目标看板`

## 6. 视觉系统

### 6.1 色彩

视觉色系采用 `骨白 + 墨灰 + 铜棕`。

- 背景：浅骨白，适合桌面端长时间使用
- 正文：深墨灰，增强信息对比
- 强调：铜棕或深琥珀，保留角色温度
- 状态色：语义化但低饱和

### 6.2 字体

采用中文优先的标题与正文字体体系。

原则：

- 标题负责气质
- 正文负责效率
- 避免中英文衬线风格混搭造成节奏破碎

### 6.3 面板层级

不再所有面板共用同一套视觉权重，而是分成三类：

- 主面板：对话控制台，层级最高
- 侧栏面板：状态信息，层级中等
- 看板面板：目标推进，强调分组和轨迹

### 6.4 动效

只保留少量有意义的过渡：

- 页面进入时分区渐进出现
- 卡片 hover 轻微抬升
- 消息发送、状态更新、计划变化使用短时过渡

动画必须服务信息变化，不做装饰性表演。

## 7. 错误与空状态

错误提示按区域归位：

- 聊天失败显示在对话控制台
- 轮询失败显示在指挥栏或当前状态卡
- 目标操作失败显示在目标看板附近

空状态统一中文化：

- `尚无对话记录`
- `尚无目标`
- `内在状态加载中`
- `还没有形成自我叙事`

## 8. 响应式约束

本次以桌面端为主，但不能在中等宽度下崩坏。

布局要求：

- `>= 1024px`：完整三段式控制台
- 中等宽度：状态侧栏折叠到对话区下方
- 更小宽度：目标看板保持单列，不出现横向滚动

## 9. 实现边界

本次只改桌面端前端展示层，不改后端接口和状态模型。

主要改动范围：

- `apps/desktop/src/App.tsx`
- `apps/desktop/src/index.css`
- `apps/desktop/src/components/*.tsx`
- `apps/desktop/src/**/*.test.tsx`

不在本次范围内：

- 接口字段新增或变更
- 数据结构重构
- 后端业务逻辑调整

## 10. 验收标准

重做完成后，应满足以下条件：

- 桌面端主信息架构改为指挥栏 + 双栏工作区 + 目标看板
- 整个 UI 全中文
- 聊天区成为主工作区，状态信息退到侧栏
- goals 由列表升级为更有推进感的任务看板
- 样式整体具备控制台感，而不是陪伴页或普通后台感
- 现有核心交互与测试能够继续通过
