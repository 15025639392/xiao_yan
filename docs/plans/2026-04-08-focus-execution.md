## 3. 只做三件大事

### 大事 A：分来源阈值配置化

目标：让 `user/world/chain` 阈值可通过配置和 API 管理，而不是只能靠默认常量。

验收标准：
- 配置层支持三类来源的 `min_score/defer_score`。
- 运行时可读到并生效，且具备合法性校验（`defer <= min`）。
- 提供回滚到上一个版本阈值的能力与记录。

### 大事 B：周度回放评估固化

目标：形成统一评估入口，保证参数变更前有可比证据。

验收标准：
- 有固定脚本/命令产出“旧参数 vs 候选参数”对比结果。
- 输出包含至少：`admit/defer/drop/wip_blocked/deferred_queue_size`。
- 每次参数变更都能留下报告文件或结构化记录。

### 大事 C：小流量上线与回退 Runbook

目标：把“可上线”变成“可控上线”。

验收标准：
- 有明确的小流量窗口、观察指标、放量条件、回滚条件。
- 线上异常能在分钟级切回安全配置。
- 值班/责任边界在文档中明确。
