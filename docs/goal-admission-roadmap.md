# 目标准入四阶段路线图（含退出条件）

> 目标：把“目标看板降噪”从规则止血演进到可控、可持续优化的最终形态。  
> 设计原则：先可观测，再拦截；先稳定，再优化；先护栏，再智能。

---

## 阶段总览

| 阶段 | 名称 | 核心策略 | 默认模式 |
|---|---|---|---|
| Phase 1 | Shadow Baseline | 打分不拦截，建立基线 | `shadow` |
| Phase 2 | Enforce Gate | 规则准入 + WIP 硬限制 | `enforce` |
| Phase 3 | Calibrated Enforce | 分来源/分场景参数校准 | `enforce` |
| Phase 4 | Hybrid Intelligence | 规则护栏 + LLM评审 + 数据闭环 | `enforce`（混合决策） |

---

## Phase 1: Shadow Baseline

### 目标
- 建立稳定、可信的观测基线，验证打分信号是否有区分度。

### 关键动作
- 开启 `GOAL_ADMISSION_MODE=shadow`。
- 采集 `admit/defer/drop/wip_blocked/deferred_queue_size`。
- 记录日级指标：新增目标数、active 平均数量、abandoned 比例。

### 退出条件（全部满足才进入 Phase 2）
- [ ] 连续 3 天统计可用，无缺失、无异常重置。
- [ ] `defer/drop` 分布有明显区分（非全 0/全 1）。
- [ ] 样本抽查中，推荐 `drop` 的噪声命中率达到预期（建议 >= 70%）。
- [ ] 出现问题可通过配置一键回退（`shadow/off`）并验证成功。

---

## Phase 2: Enforce Gate

### 目标
- 真正拦截低质量候选，把看板噪声显著压下去。

### 关键动作
- 切换 `GOAL_ADMISSION_MODE=enforce`。
- 保持 WIP 限制（如 `GOAL_WIP_LIMIT=2`）。
- 启用 defer 队列重试，保证“暂缓而非永久丢失”。

### 退出条件（全部满足才进入 Phase 3）
- [ ] 连续 7 天稳定运行，无严重误拦截事故。
- [ ] 相比基线，目标创建量下降 >= 40%。
- [ ] `deferred_queue_size` 无持续单边增长（无堆积失控）。
- [ ] 手动恢复/推进流程无回归（核心用例通过）。

---

## Phase 3: Calibrated Enforce

### 目标
- 在拦截能力稳定的前提下，降低误杀并提升命中率。

### 关键动作
- 分来源调参：`user_topic`、`world_event`、`chain_next` 使用不同阈值。
- 分场景调参：按时段/活跃度/上下文噪声水平做参数模板。
- 建立周度回放评估：用历史事件回放比较“当前参数 vs 候选参数”。

### 退出条件（全部满足才进入 Phase 4）
- [ ] 连续 2 周，误杀率持续下降或稳定在目标区间。
- [ ] `wip_blocked` 与业务反馈一致（无系统性“进不来”问题）。
- [ ] 新参数上线具备标准流程：回放评估 -> 小流量 -> 全量。
- [ ] 关键指标波动可解释（有复盘记录与参数变更记录）。

### 开工状态（2026-04-08）

- [x] 分来源阈值配置化（`user_topic/world_event/chain_next`）
- [x] 周度回放对比脚本（baseline vs candidate）
- [x] 小流量上线与回退 Runbook（见 `docs/runbooks/goal-admission-phase3-canary.md`）

---

## Phase 4: Hybrid Intelligence（最终形态）

### 目标
- 构建“可控智能准入”：规则负责边界，模型负责语义，数据负责迭代。

### 关键动作
- 引入 LLM 评审层（如 GPT-5.4），输出结构化评分与理由。
- 保留硬规则护栏：WIP、去重、速率限制、重试上限。
- 建立反馈闭环：人工标注 + 在线行为信号 + 离线评估。
- 逐步启用“模型建议权重”，但保留一键降级到纯规则。

### 最终验收条件
- [ ] 噪声目标维持低位且稳定，未出现明显策略漂移。
- [ ] 成本、延迟、准确性达到可接受平衡（有明确 SLO）。
- [ ] 线上问题可在分钟级回退到规则模式。
- [ ] 准入策略具备持续迭代机制（周度评估 + 月度优化）。

---

## 跨阶段统一治理要求

- 必须可回滚：任何阶段都能快速切回 `shadow`。
- 必须可观测：指标、日志、样本回放缺一不可。
- 必须小步推进：参数与策略变更都走“先验证后放量”。
- 必须有责任边界：产品、工程、值班角色明确。
