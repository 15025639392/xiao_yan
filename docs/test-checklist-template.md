# xiao_yan 提交前测试清单模板

本文档用于给 `xiao_yan` 的单次改动提供一个可直接复用的提交前测试清单模板。

适用场景：

- 后端主链路改动
- 前端关键页面改动
- bug 修复
- 可选依赖边界调整
- 大文件拆分
- skill / 文档 / 验证命令同步

如果你不确定这轮该测什么，可以先从本模板开始，再按实际改动删减。

## 使用方式

建议在每轮改动收尾时至少写一次下面这份清单。

可以写在：

- 提交说明
- PR 描述
- 设计文档收尾部分
- 最终交付说明

## 最小使用规则

如果时间很紧，至少不要省略以下几项：

- 入口文件
- 依赖链
- 现有测试状态
- 实际执行命令
- 剩余风险

## 项目专用建议

### 改 `services/core`

优先写清：

- 改的是哪条 `api -> service -> repository / gateway / state` 链
- 现有 `pytest` 是否覆盖到这条链
- 是否触达 `persona / memory / goals / world / tools / capability`

### 改 `apps/desktop`

优先写清：

- 改的是哪个 page 或容器
- 页面是否仍会请求关键数据
- 关键 tab / mode / button 行为是否仍成立

### 改 `tools/skills`

优先写清：

- skill 的触发条件是否还成立
- 推荐顺序是否仍和当前仓库结构匹配
- 关联文档与验证命令是否过期

## 推荐验证命令示例

按需选择，不要求每次都全跑：

```bash
cd /Users/ldy/Desktop/work/xiao_yan/services/core && pytest
cd /Users/ldy/Desktop/work/xiao_yan/apps/desktop && npm test
cd /Users/ldy/Desktop/work/xiao_yan && python3 tools/check_file_budgets.py
```

## 与 skill 的关系

如果这轮改动明显属于某个 project skill 的典型场景，建议在清单里顺手注明：

- 使用了哪个 skill
- 该 skill 对应的最低测试义务是否已满足

这样更利于后续 AI 和人类接手。
